# 프로젝트 1: AI Image Generation Platform

## 1. 프로젝트 개요

프로젝트명: AI Image Generation Platform
개발 기간: 2025년 09월 ~ 12월
개발 환경: Python 3.10, Windows 11, RTX 3060 12GB

**목적**
ComfyUI 기반의 다중 AI 모델을 활용한 이미지 생성 웹 플랫폼 개발 및 모델 품질 비교 테스트

## 2. 시스템 구조

```
Streamlit (Frontend) → FastAPI (Backend) → ComfyUI (AI Engine) → PyTorch Model
```

## 3. 주요 기능

### 3.1 이미지 생성 (Text-to-Image)
- FLUX.1 schnell, SDXL Base, SDXL Lightning 모델 지원
- 한국어 자동 번역 (Korean → English)
- 모델별 최적 파라미터 자동 적용
- 프롬프트 히스토리 관리

### 3.2 이미지 변환 (Img2Img)
- 참조 이미지 기반 스타일 변환
- Denoise 강도 조절 (0.0 ~ 1.0)
- Before/After 비교 뷰

### 3.3 이미지 업스케일
- RealESRGAN 모델 (2배/4배 확대)
- 고해상도 변환

### 3.4 이미지 분석
- BLIP 모델 기반 자동 캡셔닝
- 영어 설명 생성

### 3.5 갤러리
- SQLite 기반 히스토리 관리
- 프롬프트 검색 기능
- 통계 대시보드 (생성 수, 평균 시간, 인기 모델)

### 3.6 모델 비교
- 최대 4개 모델 동시 벤치마킹
- 성능 지표 측정 (생성 시간, img/s)
- 시스템 리소스 모니터링

## 4. 기술 스택

**AI/ML**
- ComfyUI (워크플로우 엔진)
- PyTorch 2.0+ (딥러닝)
- FLUX.1, SDXL (이미지 생성)
- BLIP (이미지 분석)
- RealESRGAN (업스케일)

**Backend**
- FastAPI 0.104.1 (REST API)
- SQLite (데이터베이스)
- Pydantic (데이터 검증)

**Frontend**
- Streamlit 1.28.2 (웹 UI)
- Pillow (이미지 처리)

**Utilities**
- deep-translator (한/영 번역)
- psutil (시스템 모니터링)

## 5. 데이터베이스 구조

**image_generations 테이블**
- id (PRIMARY KEY)
- prompt (프롬프트)
- model_name (모델명)
- steps (샘플링 스텝)
- guidance_scale (CFG)
- generation_time (생성 시간)
- image_path (이미지 경로)
- created_at (생성 일시)

**image_analysis 테이블**
- id (PRIMARY KEY)
- description (분석 결과)
- image_path (이미지 경로)
- created_at (분석 일시)

## 6. API 엔드포인트

**이미지 생성**
- `POST /comfyui/generate` - Text-to-Image
- `POST /comfyui/img2img` - Image-to-Image
- `POST /comfyui/upscale` - 업스케일
- `POST /comfyui/flux` - FLUX 모델 생성

**이미지 분석**
- `POST /describe_image` - BLIP 캡셔닝

**히스토리 관리**
- `GET /history/generations` - 생성 목록
- `GET /history/stats` - 통계 정보
- `DELETE /history/generation/{id}` - 삭제

**시스템**
- `GET /health` - 헬스 체크
- `GET /comfyui/status` - ComfyUI 상태
- `GET /comfyui/models` - 모델 목록

# 프로젝트 2: DocAI - AI 문서 생성/요약 플랫폼

## 1. 프로젝트 개요

프로젝트명: DocAI (AI Document Platform)
개발 기간: 2025년 01월 ~ 02월
개발 환경: Java 21, Node.js 18+, Windows 11

**목적**
GPT 기반 문서 자동 생성 및 요약 웹 플랫폼 개발, 외부 자동화 도구(n8n) 연동

## 2. 시스템 구조

```
React (Frontend) → Spring Boot (Backend) → MySQL + Supabase (Database)
                                        → OpenAI GPT API (AI Engine)
                                        → n8n (Workflow Automation)
```

## 3. 주요 기능

### 3.1 사용자 인증
- JWT 기반 로그인/회원가입
- 토큰 자동 갱신
- 보안 비밀번호 암호화 (BCrypt)

### 3.2 문서 업로드 & 자동 요약
- 지원 형식: .txt, .docx, .pdf
- GPT 기반 자동 요약 생성
- 문서 타입 자동 감지

### 3.3 AI 문서 생성
- 템플릿 기반 문서 생성 (보고서, 이메일, 제안서 등)
- 사용자 맞춤 프롬프트 지원
- 실시간 생성 결과 표시

### 3.4 토큰 사용량 대시보드
- 총 사용량 / 오늘 / 이번 달 통계
- 작업 타입별 사용량 (생성 vs 요약)
- 일별 사용량 차트 (최근 7일)
- 예상 비용 계산 (1,000토큰 = 약 7원)

### 3.5 API Key 관리
- 외부 서비스 연동용 API Key 발급
- 만료 기간 설정 (30일/90일/1년/무제한)
- 사용자당 최대 5개 제한
- 마지막 사용 시간 추적

### 3.6 외부 연동 (Public API)
- n8n, Zapier 등 자동화 도구 연동
- RESTful API 제공
- API Key 인증 방식

## 4. 기술 스택

**Frontend**
- React 18.x (UI 프레임워크)
- Vite 5.x (빌드 도구)
- TailwindCSS 3.x (스타일링)
- Axios 1.x (HTTP 클라이언트)
- React Router 6.x (라우팅)

**Backend**
- Spring Boot 4.0.1 (웹 프레임워크)
- Java 21 (언어)
- Spring Security 6.x (인증/인가)
- JPA/Hibernate (ORM)
- Lombok (보일러플레이트 제거)

**Database**
- MySQL (사용자, 메타데이터, API Key, 토큰 사용량)
- Supabase (문서 본문, 요약 내용)

**External Services**
- OpenAI GPT-4 (문서 생성/요약)
- n8n (워크플로우 자동화)

## 5. 데이터베이스 구조

**users 테이블**
- user_id (PRIMARY KEY)
- email (이메일)
- password (암호화된 비밀번호)
- username (사용자명)
- created_at (생성일)

**documents 테이블**
- document_id (PRIMARY KEY)
- user_id (FOREIGN KEY)
- title (제목)
- document_type (문서 타입)
- status (상태)
- upload_date (업로드일)

**api_keys 테이블**
- id (PRIMARY KEY)
- user_id (FOREIGN KEY)
- key_value (API Key 값)
- name (키 이름)
- is_active (활성 상태)
- expires_at (만료일)
- last_used_at (마지막 사용일)

**token_usage 테이블**
- id (PRIMARY KEY)
- user_id (FOREIGN KEY)
- operation_type (작업 타입)
- prompt_tokens (입력 토큰)
- completion_tokens (출력 토큰)
- total_tokens (총 토큰)
- created_at (생성일)

## 6. API 엔드포인트

**인증 API**
- `POST /api/auth/register` - 회원가입
- `POST /api/auth/login` - 로그인

**문서 API**
- `GET /api/documents` - 문서 목록
- `POST /api/documents/upload` - 파일 업로드
- `GET /api/integrated/documents/{id}` - 문서 상세
- `DELETE /api/documents/{id}` - 문서 삭제

**문서 생성 API**
- `POST /api/generation/generate` - AI 문서 생성

**API Key 관리**
- `GET /api/keys` - 키 목록
- `POST /api/keys` - 키 생성
- `DELETE /api/keys/{id}` - 키 삭제

**Public API (외부 연동)**
- `POST /api/v1/generate` - 문서 생성
- `POST /api/v1/upload` - 파일 업로드
- `POST /api/v1/summarize` - 텍스트 요약
- `GET /api/v1/health` - 헬스체크
- `GET /api/v1/auth/verify` - 인증 확인

**토큰 사용량 API**
- `GET /api/token-usage/stats/{userId}` - 사용량 통계
